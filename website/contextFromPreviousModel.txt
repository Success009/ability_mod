4. The "Handshake" Security Plan (Current Goal)
We have designed a foolproof system to prevent identity theft in chat:
In-Game: Player types /link. Plugin generates a 6-digit code and stores it in a hidden temp_codes node.
Web: User enters the code. Website writes the code to link_requests/$uid.
The Handshake: The Plugin listens to link_requests, verifies the code against temp_codes, and if it matches, moves the player’s data into the permanent security/users node.
Enforcement: Firebase Security Rules (already written) force any Web User not in the users node to appear as "Anonymous" and block them from using any other name.
